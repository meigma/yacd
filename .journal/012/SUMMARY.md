## Goal
Make `CardanoNetwork` publish enough exact non-secret localnet artifacts for a
future `CardanoDBSync` follower-node controller to join the same generated
chain as the primary node. Keep the work prototype-sized and avoid putting
verbose connection files directly in CR status.

## Outcome
The goal was met. The work landed in five GitHub squash merges: PR #18 added
the artifact publisher to the `cardano-testnet` tools image, PR #19 released
`11.0.1-yacd.2`, PR #21 fixed stale-key pruning in the publisher, PR #22
released `11.0.1-yacd.3`, and PR #20 added the controller/API integration.
Local `master` was fast-forwarded to `9ac60de`, the dev stack was stopped, and
the session implementation worktrees were removed.

## Key Decisions
- Use a same-namespace ConfigMap for artifact bytes -> CR status now carries a
  small reference and hash while verbose genesis/config/topology files stay in a
  Kubernetes data object.
- Publish exact bytes from the localnet init container -> this avoids
  re-deriving files that might not exactly match what `cardano-testnet
  create-env` produced.
- Give the init container a narrow publisher identity -> the primary Deployment
  uses a dedicated ServiceAccount with pod token automount disabled, and only
  the init container receives a projected token for a Role limited to its
  network artifact ConfigMap.
- Make `ArtifactsReady` a real readiness gate -> dependent controllers should
  only trust `status.artifacts` after the controller verifies schema version,
  localnet fingerprint, allowed keys, and the live data hash.
- Recover corrupted artifact ConfigMaps by delete-and-republish -> changing the
  ConfigMap UID rolls the primary Pod so the init path can republish exact
  generated bytes.
- Delete and return during recovery -> Kubernetes deletion is asynchronous, so
  the controller now waits for a later reconcile to recreate the same-name
  ConfigMap instead of assuming immediate name reuse.
- Document manager RBAC as a trust boundary -> the per-network publisher Role is
  narrow, but the manager still runs with cluster-scoped watch/write RBAC for
  this local-first prototype.

## Open Threads
- Implement the `CardanoDBSync` controller and consume
  `CardanoNetwork.status.artifacts.networkConfigMapName` when building the
  follower-node localnet configuration.
- Namespace-scoped manager mode remains a future hardening path if YACD needs to
  be safe outside a trusted dev or team cluster.
- Public/custom network artifact publishing is still deferred; this slice is
  local-mode only.
- The artifact publisher only patches ConfigMaps. If the bundle grows near
  Kubernetes object size limits, move the artifact payload behind a different
  storage/reference boundary.

## Lessons
- The first Kind smoke caught a projected-token audience bug that unit/envtest
  could not see. For init-container API calls using Kubernetes projected
  service account tokens, in-cluster smoke remains valuable.
- Hash annotations prove consistency only after the controller recomputes the
  hash over live data and rejects keys outside the schema allowlist.
- Same-name Kubernetes recovery should assume deletion can be asynchronous,
  especially once finalizers or slow API deletion are involved.
